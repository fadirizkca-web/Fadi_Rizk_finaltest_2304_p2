# Reflection: Version Control and Project Management

# Workflow and Repository Management

1- My workflow for this project followed a structured approach to ensure code integrity and clear documentation. I began by initializing the repository with a clear README to establish a "Source of Truth" for the project from the very first commit. By cloning the repository locally using **"git clone"**, I was able to manage file updates in a controlled environment before pushing changes back to the main branch. I utilized issues to break down the project requirements into manageable tasks, ensuring that every update, such as adding the web app files and templates, was tracked and documented.The Value of Milestones

2- Integrating milestones into my workflow provided a critical high-level view of the project's timeline. By naming the milestone Fadi_Rizk_ft_milestone_1 and setting a firm due date, I created a sense of urgency and a clear target for the "first working version". The primary value of using milestones is the ability to link multiple issues to a single goal, which allows for better progress tracking and ensures that no technical requirements are overlooked during the development cycle. This organized structure not only assists in meeting deadlines but also improves the transparency of the development process for all stakeholders.
